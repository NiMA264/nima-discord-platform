# GitHub Project Setup

Use GitHub Projects (v2) with these columns:
- Backlog
- Ready
- In Progress
- Review
- Done

Suggested custom fields:
- `Priority` (S, A, B)
- `Area` (Core, Commands, Integrations, Dashboard, AI, Ops)
- `Risk` (Low, Medium, High)

Recommended first items:
1. Alpha stabilization bug triage
2. Test-guild onboarding feedback
3. Notification quality tuning
4. Dashboard v1 polish

Automation suggestion:
- New issues -> `Backlog`
- PR opened -> `Review`
- PR merged -> `Done`
